# (WIP) Mable Code Conventions
This document is not complete; it's just somewhere for me to jot down stuff
so I don't violate my own conventions for now (lol).

- Everything that can be `final` should be `final` unless it has a very good reason
- Class field declarations should always go before its first reference.
- Prefer composition over hack-ish methods/functions that make edits to classes.
